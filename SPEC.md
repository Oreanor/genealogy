# Родословная семьи Никонец — Техническая спецификация

## 1. Общая концепция

Веб-приложение в виде интерактивного альбома-книги на Next.js (App Router, последняя версия).
Пользователь листает "развороты книги", каждый разворот — левая и правая страницы.

---

## 2. Первый экран (титульный разворот)

**Левая страница:**
- Название: «Родословная семьи Никонец»
- Поле для большой фотографии (обложка/титул)

**Правая страница — оглавление:**
- Семейное древо
- Персоны
- История
- Фото
- Другие материалы

Каждый пункт ведёт на первый разворот соответствующей главы.

---

## 3. Навигация

- При переходе из оглавления на главу — роутинг на главу (например `/chapter/family-tree`, `/chapter/persons` и т.д.)
- В верхней части экрана (или книги) — закладка «Оглавление», всегда видимая при просмотре глав, ведёт обратно на титульный разворот
- Каждая глава может состоять из нескольких разворотов (листание вперёд/назад внутри главы)

---

## 4. Структура контента

### 4.1. Массив страниц (разворотов)

Каждая страница — объект с поддержкой гипертекста:

```ts
// Блок контента — текст с форматированием и ссылками
type ContentBlock = {
  type: 'paragraph' | 'heading' | 'list';
  content: RichTextNode[];  // inline: text, bold, italic, link
};

type RichTextNode = 
  | { type: 'text'; value: string }
  | { type: 'bold'; children: RichTextNode[] }
  | { type: 'italic'; children: RichTextNode[] }
  | { type: 'link'; href: string; children: RichTextNode[] };

type Page = {
  id: string;
  chapter: ChapterId;
  spreadIndex: number;  // порядок разворота в главе
  left: PageContent;
  right: PageContent;
};

type PageContent = {
  blocks?: ContentBlock[];
  image?: ImageConfig;
};
```

### 4.2. Интерактивные фотографии

- Фото может содержать несколько «горячих зон» (кликабельных областей)
- Область задаётся: многоугольник (polygon) или прямоугольник (rect)
- При клике — переход в главу «Персоны» на карточку выбранного человека

```ts
type ImageConfig = {
  src: string;
  alt?: string;
  hotspots?: Array<{
    personId: string;  // id персоны из данных
    coords: number[];  // [x1,y1, x2,y2, ...] в процентах 0-100
    shape: 'rect' | 'polygon';
  }>;
};
```

## 5. Структура данных семейного древа

### Минимальный и достаточный набор

**Для учёта братьев и построения дерева достаточно только связи родитель–ребёнок.**
Братья автоматически выводятся как персоны с общими родителями.

### Персона

```ts
type Person = {
  id: string;
  name: string;
  birthYears?: string;      // напр. "1925–1998" или "ок. 1890"
  birthPlace?: string;
  occupation?: string;
  parentIds: string[];      // id родителей (0, 1 или 2)
};
```

- `parentIds`: если пусто — корень дерева; 1–2 элемента — родители
- Дети выводятся: у кого в `parentIds` указан этот человек — те и есть дети
- Братья/сёстры: персоны с одинаковым непустым набором `parentIds`

### Расширение при необходимости

Если понадобятся супруги:
```ts
type Person = {
  // ...
  spouseId?: string;  // связь с супругом (опционально)
};
```

Для старта — только `parentIds` достаточно.

## 6. Главы и содержимое

| Глава | Описание |
|-------|----------|
| Семейное древо | Визуализация дерева по `Person[]`, клик на персону → глава Персоны |
| Персоны | Карточки людей (имя, фото, годы, место, род занятий) |
| История | Развороты с текстом, фото, ссылками |
| Фото | Галерея/развороты с интерактивными фото |
| Другие материалы | Любые дополнительные развороты |

## 7. Разбивка на задачи

### Фаза 1: База проекта
1. **Инициализация Next.js** — создать проект (create-next-app), настроить TypeScript, базовые стили
2. **Базовая вёрстка «книги»** — компонент разворота (левая/правая страница), общий layout

### Фаза 2: Навигация и оглавление
3. **Титульный разворот** — левая: заголовок + фото, правая: оглавление с ссылками
4. **Роутинг глав** — маршруты `/`, `/chapter/[slug]`, закладка «Оглавление» в шапке
5. **Листание внутри главы** — навигация вперёд/назад по разворам одной главы

### Фаза 3: Контент и гипертекст
6. **Компонент RichText** — рендер bold, italic, link из структуры данных
7. **Редактор/поддержка разметки** — (опционально) ввод разметки для ссылок, жирного, курсива (если нужно редактирование)
8. **Массив страниц и данные** — типы, мок-данные, загрузка контента по главам

### Фаза 4: Интерактивные фото
9. **Компонент ImageWithHotspots** — изображение + overlay с областями (rect/polygon)
10. **Привязка к персонам** — клик по зоне → переход в главу Персоны на нужного человека

### Фаза 5: Древо и персоны
11. **Модель данных Person** — типы, пример JSON, загрузка ✅
12. **Визуализация дерева** — отображение по `Person[]` (parentIds), раскрытие уровней ✅
13. **Страница/карточка персоны** — разворот или модалка с данными персоны ✅
14. **Связка древо → персона** — клик по узлу → переход на карточку персоны ✅

### Фаза 6: Полировка
15. **Анимации перелистывания** — (опционально) эффект страниц
16. **Адаптивность** — работа на планшете (мобила не требуется) ✅
17. **Сборка и деплой** — проверка prod-сборки, подготовка к хостингу ✅

## 9. Стек и зависимости (предложение)

- **Next.js** (latest) — App Router, React Server Components где уместно
